# Security Policy

感谢你帮助 Ech0 提升安全性。

## Supported Versions

我们通常只对以下版本提供安全修复支持：

- 最新发布版本（Latest）
- 上一个稳定版本（Previous）

更早版本可能不会收到安全补丁，建议尽快升级。

## Reporting a Vulnerability

请不要在公开 Issue 或 Discussion 中披露安全漏洞细节。

推荐优先使用 GitHub 私密漏洞报告（Security Advisories）：

- https://github.com/lin-snow/Ech0/security/advisories/new

如果你无法使用上面的方式，请通过维护者可控的私密渠道联系，并包含以下信息：

- 漏洞类型与影响范围
- 复现步骤或 PoC
- 受影响版本、环境与配置
- 可能的修复建议（可选）

## Response Process

收到报告后，我们会尽力按以下节奏处理：

- 48 小时内确认收到
- 7 天内完成初步评估与分级
- 修复后尽快发布版本并更新公告

具体时长会受漏洞复杂度与发布窗口影响。

## Coordinated Disclosure

在补丁发布前，请避免公开披露漏洞细节。

我们鼓励遵循负责任披露流程，在修复发布后再同步公开技术细节，以保护用户安全。
