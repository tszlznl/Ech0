你是否也希望：别人点进你的站点，**第一眼就知道这是你的地方**；而你发帖、评论时，**头像和名字始终一致**？在 Ech0 里，「整座站长什么样」和「你这个账号长什么样」是分开的——**前者**给访客与互联场景一个稳定的门面，**后者**让你在时间线上被认出来。下面按这个逻辑写，尽量少列菜单路径，多写「它到底在帮你什么」。

---

## 先弄清：你要改的是「空间」，还是「自己」？

后台侧边栏里，和外观、身份相关的入口主要有两处：

| 你想动的是… | 去哪里 |
| ----------- | ------ |
| 全站 Logo、标题、服务地址、页脚、默认语言、开放注册等 | **偏好设置**（`/panel/setting`）里的 **系统设置** |
| 自己的头像、用户名、密码、邮箱、界面语言 | **用户中心**（`/panel/user`）里的 **用户设置** |
| 脚本 / API 用的令牌（与长相无关） | 同一页 **偏好设置** → **访问令牌** |

系统设置里点 **编辑**，改完点 **应用**；用户中心同理。

---

## 偏好设置 → 系统设置：这座站在对外说什么？

这里决定的是 **未登录访客**（以及互联、元信息）看到的世界，不是某一位用户的偏好。

1. **站标，即第一印象** — 上传的站点 Logo 会作为全站图标出现；**未登录**时常在首页顶栏展示。**已登录**时，顶栏往往会换成 **你的个人头像**，所以别忘了单独设站标——否则访客眼里这座站会少了「一枚徽章」。图片走与附件相同的存储（本地或 [对象存储](/docs/guide/s3)）。

2. **站点标题与服务名称** — 标题多出现在标签页、SEO；服务名称多用在时间线、互联等**展示名**。可以一强一弱、一正式一亲切，也可以相同，按你的品牌习惯来。

3. **服务地址，你在互联网上的门牌** — 填对外访问的完整 URL（带 `https://`）。互联、他站拉取你的头像或实例信息时，常会依赖这个地址；填错或换域名后未更新，会出现「别人看不到你头像」之类问题。详见 [常见问题](/docs/start/faq) 与 [互联](/docs/guide/federation)。

4. **页脚：一句落款** — 文案与可选链接（备案、关于页等），给页面一个安静的收尾。

5. **Meting API：音乐卡片背后的解析** — 填写可信的 Meting 服务地址；生产环境更建议自建或可控端点，少把关键体验绑在陌生服务上。

6. **自定义 CSS / Script：只给懂你的人** — 样式用来微调气质，脚本能力更强，但会影响所有访客页面。**仅应由可信管理员** 编辑；恶意或错误脚本可能带来异常甚至安全风险（XSS），改前请备份。

7. **默认语言与开放注册** — 默认语言影响访客、以及尚未在个人里改语言的用户；**允许注册**则决定是否开放自助注册（其余策略以当前版本为准）。

---

## 偏好设置 → 访问令牌

令牌给自动化、脚本、[MCP](/docs/guide/mcp) 用，不负责「长什么样」。需要时请看 [访问令牌](/docs/guide/accesstoken)。

---

## 用户中心：你是谁？

这里只影响 **当前登录账号**，不会替别人改全站。

1. **头像，被认出来的方式** — 出现在评论、个人相关展示、以及登录后顶栏等处。上传后在界面里 **应用** 保存，具体以当前版本提示为准。

2. **用户名、密码、邮箱** — 登录与展示用；密码留空是否表示「不修改」以界面为准；邮箱用于联系或找回（若已启用相关能力）。

3. **界面语言，只属于你的界面** — 只影响你自己看到的语言，**不会**改掉全站在「系统设置」里设的**默认语言**。

## 一眼对照

| 你想改… | 去哪里 |
| ------- | ------ |
| 站标、全站标题、服务地址、页脚、Meting、自定义样式/脚本、默认语言、开放注册 | **偏好设置** → 系统设置 |
| 个人头像、自己的账号资料、自己的界面语言 | **用户中心** → 用户设置 |
| API 令牌 | **偏好设置** → 访问令牌 |

若界面文案与本文不一致，**以你实例上的界面与 OpenAPI（`/swagger/index.html`）为准**。
