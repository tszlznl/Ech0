title: AI 模型与近期摘要
description: 配置 AI 生成模型（Agent），它驱动近期摘要与 AI 问答
---

Ech0 的各项 AI 能力**共用同一个「生成模型」配置**。在 **系统设置 → Copilot → 模型** 中填好一次，即可同时驱动：

- **近期摘要**：由大模型根据你最近的 Echo 生成一段简短总结，便于访客快速了解动态（本页下半部分）。
- **[AI 问答（Chat）](/docs/guide/chat)**：用自然语言跟你自己的 Echo 对话。

如果还想让问答按「语义」检索历史内容，可再配一份[向量检索](/docs/guide/embedding)（可选）。

---

## 配置项

| 项 | 说明 |
| --- | --- |
| 启用 Agent | 总开关。关闭后近期摘要与 AI 问答都不可用 |
| 接口协议 | **仅两项**：**OpenAI 兼容**、**Anthropic** |
| 模型名称 | 如 `gpt-4o`、`deepseek-chat`、`claude-3-5-sonnet` 等，以厂商文档为准 |
| API Key | 访问密钥（勿泄露） |
| Base URL | 自建或兼容网关地址；接 DeepSeek、Qwen、Ollama 等服务时在此填其接口地址 |
| Prompt | 自定义系统提示。**仅作用于「近期总结」**，AI 问答不会使用它 |
| 多模态支持 | 开启后，**AI 问答**检索命中带图片的 Echo 时会把配图一并交给模型理解（需所配模型支持图片输入） |
| 上下文窗口 | 填 `256k`、`1m` 等，留空默认 `256k`。窗口越大越倾向一次性读入全部内容，越小越早转为分层总结 |

::: tip 选的是协议，不是厂商
这里选的是**接口协议**而不是具体厂商。DeepSeek、Qwen、通义、Ollama、各类自建兼容网关都走 **OpenAI 兼容** 协议——选它，再用 `Base URL` + `模型名称` 指定具体实现即可。Anthropic（Claude）走自己的原生协议。
:::

具体字段名以界面为准。

## 近期摘要

启用后，Ech0 会从你**近期的 Echo** 抽取上下文，调用模型生成一段简短总结对外展示。

- 为避免短时间重复打爆 API，摘要会**缓存**；新发 Echo 会让缓存失效、下次访问时重新生成。
- 摘要风格与长度可通过上面的 **Prompt** 调整。

### HTTP 接口（节选）

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| GET | `/api/agent/recent` | 获取对外展示的近期摘要 |
| GET | `/api/agent/info` | 查询 Agent 是否启用等元信息 |
| GET/PUT | `/api/agent/settings` | 读取或更新配置（需登录） |

完整定义见 **`/swagger/index.html`**。
