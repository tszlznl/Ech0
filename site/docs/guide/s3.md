启用后，附件可写入 **S3 兼容对象存储**（AWS S3、MinIO、Cloudflare R2、阿里云/腾讯云等兼容 S3 API 的服务），减轻本机磁盘压力并便于做 CDN。

---

## 启用前准备

向云厂商或自建服务准备好：

1. **桶（Bucket）名称**  
2. **Access Key / Secret Key**（或厂商等价凭证）  
3. **Endpoint**：厂商给出的 API 接入地址（如 `s3.ap-southeast-1.amazonaws.com` 或 MinIO 地址）  
4. 若用 **AWS 区域型 Endpoint**，通常还要填 **Region**

---

## 后台配置项（逐项说明）

### 启用 S3

总开关。关闭时走本地（或默认）存储；打开后出现下列项。

### 启用 SSL

建议**始终开启**，避免密钥与数据在链路上明文传输。

### S3 服务 / 模板

选择厂商模板（如 **AWS S3**、**MinIO**、**Cloudflare R2**、**Other**），用于预填部分默认值；选 **Other** 时需自行核对 Endpoint 格式。

### S3 Endpoint

填厂商提供的 **API Endpoint**，一般**不要**在框内写 `http://` 或 `https://` 前缀（按你界面提示为准；有的界面用开关表示 SSL）。

### Access Key / Secret Key

即 API 访问密钥对；Secret 仅自己可见，勿泄露。

### Bucket

对象存放的桶名；需事先在控制台创建好。

### Path Prefix（路径前缀，可选）

若填写 `prod/ech0`，则对象键前会统一带上此前缀，便于同桶多环境隔离。不填则直接落在桶根下对应规则路径。

### Region

使用 AWS 等需要区域的模板时填写（如 `ap-southeast-1`）；纯路径型 Endpoint 的厂商按说明填写或留空。

### CDN / 公网访问（若界面提供）

若附件需通过 CDN 或固定域名访问，按界面配置 **CDN 域名** 或 **公共读策略**；否则浏览器可能无法直接加载图片/媒体。**桶或 CDN 策略**需允许你的站点域名跨域上传（浏览器直传时出现 CORS 错误时，要在存储或网关侧配置 CORS）。

---

## 权限与策略

用于上传的密钥至少需能对目标桶执行：**列出、上传、读取、删除**（具体动作名因厂商而异）。若保存时报 **Access Denied**，请检查：

- Access / Secret 是否错误；  
- IAM/策略是否限制到错误桶或路径；  
- 桶名、Region、Endpoint 是否混用（例如把 OSS 的 Endpoint 填进 AWS 模板）。

## 与「数据管理」的关系

更换桶、迁移本地⇄对象、批量修 URL 等运维操作风险高，仓库内另有 **[存储迁移指南](https://github.com/lin-snow/Ech0/blob/main/docs/usage/storage-migration.md)**（说明 `key`、`PathPrefix`、换桶注意事项）。执行前请**完整备份**数据库与文件。

## 常见问题

**Path Prefix 必须填吗？**  
不必须；填了便于同桶多项目隔离。

**配置都对但仍无法上传？**  
打开浏览器开发者工具，看是否 **CORS** 报错；若有，需在对象存储或前置网关配置允许你的站点源。

**媒体打不开？**  
检查桶/对象的 **公共读** 或 **签名 URL** 策略，以及是否改了 Endpoint/CDN 却未同步更新库里记录的访问地址。
