Ech0 的数据主要包括：**数据库（含业务元数据）** 与 **附件文件**（本地目录或对象存储）。日常运维围绕**导出、迁移、备份、恢复**展开。

---

## 在网页里能做什么

打开 **管理面板 → 数据管理**（名称以当前版本为准），通常包括：

- **快照导出**：打包为 ZIP，便于离线归档或换机。
- **迁移导入**：从支持的来源导入历史数据（见下文）。

导出文件命名、存放路径以界面提示为准；若提供「下载链接」或「后台任务」，按向导操作即可。

---

## 命令行与 TUI

使用**二进制安装**时，可在服务器上：

```bash
ech0 tui
```

在菜单中执行备份相关操作（具体项以 TUI 为准）。

也可使用：

```bash
ech0 backup
```

（子命令以 `ech0 --help` 为准。）

适合写 **cron** 或 CI 做定时打包。

---

## 迁移来源（常见）

| 来源              | 说明                                   |
| ----------------- | -------------------------------------- |
| 其他 Ech0 v4 实例 | 按迁移向导上传/选择包                  |
| Ech0 v3           | 需先在 v3 **导出快照**，再在 v4 走迁移 |
| Memos 等          | 按向导支持的格式导入                   |

迁移过程中通常会**禁止并发写入**（写锁），避免数据错乱；完成后可选择是否合并部分系统设置（以向导为准）。

> **v3 → v4**：不能原地升级。必须：v3 导出快照 → 部署 v4 → 在 v4 使用迁移导入。

## 定时自动备份

在 **系统设置 → 备份计划** 中：

1. 打开自动备份开关。
2. 填写 **Cron 表达式**（支持常见 5 位或 6 位格式）。

示例含义（具体以你填的时区与文档为准）：

| 表达式        | 含义（示意）    |
| ------------- | --------------- |
| `0 2 * * 0`   | 每周日凌晨 2:00 |
| `30 3 * * *`  | 每天 3:30       |
| `0 */6 * * *` | 每 6 小时       |

执行结果可在日志或面板中查看；备份文件建议**定期下载到异地**。

---

## 手动恢复（高风险）

仅建议在清楚后果时操作：

1. **停止** Ech0 服务或容器。
2. **备份当前 data 目录**（整个复制一份）。
3. 从可信快照 ZIP **解压**，用其中文件**覆盖**数据目录中的数据库与文件（路径需与官方目录结构一致）。
4. 启动服务并验证。

误覆盖或版本不匹配可能导致无法启动，请务必先保留一份可回滚的备份。

---

## 与对象存储、文件路径

若已启用 **S3**，附件实际在桶内；数据库里存的是引用信息。换桶、换域名、本地⇄对象互迁等，请参考仓库 **[存储迁移指南](https://github.com/lin-snow/Ech0/blob/main/docs/usage/storage-migration.md)**，并在测试环境验证后再上生产。

---

## OpenAPI

备份、导出等相关 HTTP 接口（若开放）见实例 **`/swagger/index.html`**；调用时通常需要 **访问令牌** 或管理员会话，勿把令牌写入公开脚本。
