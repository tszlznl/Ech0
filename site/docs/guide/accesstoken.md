**访问令牌（Access Token）** 是一串可放在 HTTP 请求头里的凭证，让**脚本、命令行工具、自动化服务**以你的身份调用 Ech0 的 **REST API**，而无需在程序里保存登录密码。

在管理后台进入 **系统设置 → 访问令牌** 创建；**只有管理员**可以创建与删除令牌。界面中 **权限范围** 已按「内容 / 文件 / 互联 / 资料 / 管理」分组，与下表中的 Scope 一一对应。

---

## 这篇文档适合谁读

- 你要写 **curl / Python / n8n** 等调用 `/api/...` 接口。  
- 你要配置 **集成评论**、**MCP（AI 接入）** 等场景（不同场景要选对 **Audience**）。  
- 你不需要先懂 JWT；下面会说明**请求头怎么写**、**权限怎么选**。

---

## 基本用法

在任意需要鉴权的请求里加上（把 `<令牌>` 换成创建时复制下来的整串字符）：

```http
Authorization: Bearer <令牌>
```

- 令牌与**浏览器里登录用的 Session** 不是同一个东西；适合机器调用。  
- **具体哪些路径需要令牌、请求体字段** 以你部署实例上的 **`/swagger/index.html`** 为准（不同版本可能略有差异）。

---

## Audience（受众）是什么、怎么选

可以把 **Audience** 理解成「这枚令牌**打算给谁用**」——同一套 Scope 在不同 Audience 下，**能访问的入口可能不同**。后台下拉里四项与内部取值对应如下（以中文界面为准）：

| 界面名称 | 内部值 | 典型用途 |
| -------- | ------ | -------- |
| **公开客户端** | `public-client` | 浏览器外的一般 API 调用、自建小工具 |
| **命令行** | `cli` | 命令行或脚本（与「公开客户端」类似，按习惯区分用途） |
| **系统集成** | `integration` | **集成评论**等：`POST /api/comments/integration`（需同时勾选 `comment:write` 等，见 [评论系统](/docs/guide/comment)） |
| **MCP（AI Agent）** | `mcp-remote` | **仅**用于连接 **`/mcp`**（Cursor、Claude Desktop 等）。用错 Audience 访问 MCP 会 **403** |

**不要混用**：例如拿「公开客户端」令牌去调 `/mcp` 会被拒绝；做 AI 接入请单独建 **MCP（AI Agent）** 令牌，见 [MCP 接入](/docs/guide/mcp)。

---

## Scope（权限范围）一览

勾选时遵循**最小权限**：只给脚本真正需要的权限。下面按「能做什么」说明（与后端常量一致）。

| Scope | 含义（白话） |
| ----- | ------------- |
| `echo:read` | 读公开/有权限的时间线、单帖、标签等 |
| `echo:write` | 发布、编辑、删除自己的动态，点赞等写操作 |
| `comment:read` | 读评论列表 |
| `comment:write` | 发表评论（含集成评论路径，需配对 Audience） |
| `comment:moderate` | 评论审核、管理等（若接口要求） |
| `file:read` | 读文件元数据、下载相关接口 |
| `file:write` | 上传、删除文件等 |
| `connect:read` | 查看互联（Connect）配置与对端信息 |
| `connect:write` | 添加、删除互联 |
| `profile:read` | 读当前用户资料 |
| `admin:settings` | 系统设置、Webhook 等**管理级**接口（仅管理员账号创建的令牌可包含） |
| `admin:user` | 用户管理类接口 |
| `admin:token` | 管理访问令牌本身等 |

包含 **`admin:*`** 的令牌威力大，仅在你完全信任的运行环境里使用，且**不要**提交到 Git 或截图外传。

## 有效期

创建时在界面选择（对应后端取值）：

| 选项 | 内部值 | 说明 |
| ---- | ------ | ---- |
| 约 8 小时 | `8_hours` | 适合临时脚本、一次性任务 |
| 约 1 个月 | `1_month` | 适合个人长期自动化 |
| 长期有效 | `never` | **泄露风险最高**，仅在可控环境使用 |

过期后该令牌会失效；列表里过期记录可能被自动清理。怀疑泄露时请**立即删除**该令牌并新建。

---

## 创建与生命周期

1. 使用**管理员账号**打开 **系统设置 → 访问令牌**。  
2. 填写**名称**（方便你以后辨认用途）、**有效期**、**Audience**、**Scopes**。  
3. 创建成功后，**完整令牌字符串通常只显示一次**，请立刻复制到密码管理器或私密环境。  
4. 列表中可随时**删除**即吊销；无法「找回」同一串字符，只能新建。

## 常见场景对照

| 你想做… | Audience | Scope（至少） |
| ------- | -------- | ------------- |
| 只读拉取时间线、写脚本调 API | **公开客户端** 或 **命令行** | `echo:read` |
| 脚本发帖 | 同上 | `echo:read`、`echo:write` |
| **集成评论**（无验证码、无表单 token） | **系统集成** | `comment:write`（并参考 Swagger 与 [评论系统](/docs/guide/comment)） |
| **Cursor / MCP 客户端** 连 Ech0 | **MCP（AI Agent）** | 按 [MCP 接入](/docs/guide/mcp) 勾选 |
| 管理 Webhook、部分管理接口 | 公开客户端等 + **`admin:settings`** | 视接口而定 |

---

## 安全建议

- 把令牌当作**密码**：不要写进公开仓库、不要贴到论坛；CI 里用密钥库或环境变量。  
- 定期**轮换**：用过期时间较短的令牌，或手动删除再建。  
- **最小权限**：能只读就不要给 `write`，更不要给 `admin:*` 除非必要。  
- 若怀疑泄露：**立刻删除**对应令牌，并检查实例审计与异常操作。

## 与 OpenAPI / 版本差异

REST 路径、请求体、响应码以实例上的 **Swagger** 为准；升级 Ech0 后若调用失败，请对照新版本文档检查 Scope 与路径是否有变更。
