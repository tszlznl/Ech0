---

**AI 问答** 让你用自然语言跟自己的历史 Echo 对话。它不是凭空作答：模型会**先检索/聚合/统计你的 Echo，再据此回答**，并把命中的来源标注出来。底层是「RAG 检索 + 多轮工具调用」。

::: tip 这是给站长自己的功能
`POST /chat` 需要**管理员权限**，**只有站点 owner / 管理员能用**，访客看不到、也调不动。你的 Echo 内容不会因为这个功能对外暴露。
:::

---

## 开启前提

1. **必须**：先在 **系统设置 → Copilot → 模型** 配好[生成模型](/docs/guide/agent)（启用 Agent + 协议/模型/Key）。这是 Chat 的大脑。
2. **可选**：再配一份[向量检索](/docs/guide/embedding)。配了，问答按**语义**找历史内容（换个说法也能命中）；不配，退化为**关键词**检索，仍可用。

> 「模型」里的自定义 **Prompt 不影响 Chat**，它只作用于近期摘要。

---

## 怎么用

- **首页悬浮入口**：登录后首页右下角的对话入口，点开即问。
- **独立对话页**：进入 Chat 页面进行多轮对话。

不知道问什么，可以从这些开始：

- 「总结我最近一段时间在关注和思考什么」
- 「我这段时间的心情有什么变化」
- 「把我记过的零散想法汇总一下」
- 「我去年关于读书都写过些什么」

---

## 它能做什么

回答前，模型会按需调用以下能力（一次对话可多次、可组合）：

| 能力 | 用途 |
| --- | --- |
| 检索 Echo | 按 query（语义/关键词）+ 标签 + 日期范围找最相关的若干条 |
| 区间总结 | 聚合某时间段内的**全部** Echo，用于写「年终 / 季度 / 月度」这类跨度较长的回顾 |
| 统计概览 | 给出某区间的条数、常用标签等总体情况 |

所以「帮我写年终总结」「我九月都在忙什么」这类问题，它会自动选对应的方式去取数，而不是只采样几条。

---

## 过程是可见的

回答以**流式**返回，过程透明：

- **检索中**：显示模型正在搜什么。
- **来源引用**：列出本次命中的 Echo，可点回原文核对。
- **逐字输出**：答案边生成边显示。

若开启了**多模态支持**且模型支持图片输入，命中带图的 Echo 时会把配图一并交给模型理解。

---

## 会话与隐私

- 对话有**上下文记忆**，可随时清空会话重新开始。
- 多用户实例下，检索严格**收口到你本人**发布的 Echo，不会混入他人内容。
- 调用第三方 LLM 可能产生费用，并会把相关 Echo 片段发送给所配服务商；请遵守其条款，按需选择模型。
