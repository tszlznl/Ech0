## 模式说明

编辑器支持多种模式，通过界面上的**模式切换 / 面板**进入（名称以当前版本为准）：

| 模式   | 用途                                            |
| ------ | ----------------------------------------------- |
| 主编辑 | 撰写 Markdown 正文                              |
| 面板   | 选择扩展类型、进入收件箱等                      |
| 扩展   | 配置音乐、视频、GitHub 仓库、网站链接等卡片数据 |
| 图片   | 上传附件、选择版式（如瀑布流）、开关压缩        |
| 标签   | 为当前 Echo 添加或移除标签                      |
| 收件箱 | 查看系统通知等（若版本提供该入口）              |

从面板进入扩展后，按提示填 URL 或元数据即可生成对应卡片。

---

## 草稿

编辑过程中会**自动保存草稿**到浏览器本地，防止误关页面丢内容。草稿有**有效期**（例如约一天，以实现为准），过期会清理。

再次打开编辑器时，若存在未发送的草稿，通常会提示**恢复或丢弃**；正在编辑**已发布**的 Echo 时，行为可能与「新发一条」略有不同，以当前版本为准。

---

## 图片与上传

- 支持**拖拽**、选择文件上传。
- 可开启**智能压缩**以减小体积：Safari 常见输出 JPEG，其他浏览器多为 WebP（以实现为准）。
- 若实例配置了 **S3**，大文件可能走直传或后端代理，以存储设置与网络为准。
- 单批数量、大小限制见界面提示或 **OpenAPI**。

## 与评论、RSS
