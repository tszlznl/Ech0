一条 **Echo** 通常从「写正文」开始，再按需加图片、链接卡片、音乐等。界面文案可能随版本微调，以下描述与当前主流布局一致。

---

## 第一次发一条 Echo（流程）

1. 在界面中找到 **发布 / 新建** 入口（一般在时间线附近），打开编辑器。  
2. 在 **主编辑区** 输入 Markdown 正文；支持常见语法（标题、列表、引用等，以实现为准）。  
3. 需要配图时，切到 **图片** 相关区域，拖拽或选择文件上传（见下文「图片与上传」）。  
4. 需要音乐、视频、GitHub 仓库等「卡片」时，通过 **面板 → 扩展** 按提示填写 URL 或元数据。  
5. 确认无误后 **发布**。若站点开启评论，访客侧展示逻辑见 [评论系统](/docs/guide/comment)。  

若发布后看不到内容，先确认当前账号是否有**发帖权限**（见 [常见问题](/docs/start/faq)）。

---

## 模式说明

编辑器支持多种模式，通过界面上的**模式切换 / 面板**进入（名称以当前版本为准）：

| 模式   | 用途                                            |
| ------ | ----------------------------------------------- |
| 主编辑 | 撰写 Markdown 正文                              |
| 面板   | 选择扩展类型、进入收件箱等                      |
| 扩展   | 配置音乐、视频、GitHub 仓库、网站链接等卡片数据 |
| 图片   | 上传附件、选择版式（如瀑布流）、开关压缩        |
| 标签   | 为当前 Echo 添加或移除标签                      |
| 收件箱 | 查看系统通知等（若版本提供该入口）              |

从面板进入扩展后，按提示填 URL 或元数据即可生成对应卡片。

---

## 草稿

编辑过程中会**自动保存草稿**到浏览器本地，防止误关页面丢内容。草稿有**有效期**（例如约一天，以实现为准），过期会清理。

再次打开编辑器时，若存在未发送的草稿，通常会提示**恢复或丢弃**；正在编辑**已发布**的 Echo 时，行为可能与「新发一条」略有不同，以当前版本为准。

---

## 图片与上传

- 支持**拖拽**、选择文件上传。
- 可开启**智能压缩**以减小体积：Safari 常见输出 JPEG，其他浏览器多为 WebP（以实现为准）。
- 若实例配置了 **S3**，大文件可能走直传或后端代理，以存储设置与网络为准。
- 单批数量、大小限制见界面提示或 **OpenAPI**。

---

## 与评论、RSS、Hub

- **RSS**：单条 Echo 发布后，站点若提供 RSS，阅读器可订阅（入口以界面为准）。  
- **评论**：需在 [评论系统](/docs/guide/comment) 中开启后，访客才可见评论入口。  
- **Hub / 互联**：多实例聚合浏览见 [互联聚合](/docs/guide/federation)；单实例可忽略。  

编辑已发布内容时，请以当前版本的「编辑」入口为准；涉及媒体 URL 与存储后端变更时，注意 [对象存储](/docs/guide/s3) 与 [数据管理](/docs/guide/datacontrol) 中的说明。
