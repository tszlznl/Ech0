若你刚听说 Ech0，建议先读 [产品概览](/docs/guide/overview)，再按本页顺序操作。下面假设你有一台能装 Docker 的 Linux 服务器或本机，目标是在 **10 分钟内** 打开网页并完成首次登录。

---

## 你会得到什么

完成本节后，你将拥有：

- 一个在浏览器里可访问的 Ech0 实例（默认端口 **6277**）
- 一个 **Owner（首个注册）** 账号，可管理站点与发帖权限
- 对「数据存在哪、如何备份」有基本概念

若你更关心**概念**而不是命令，可直接跳到 [名词与界面](#名词与界面)。

---

## 部署前：你需要准备什么

| 项目         | 说明                                                                                                                       |
| ------------ | -------------------------------------------------------------------------------------------------------------------------- |
| 一台机器     | 云服务器、NAS、树莓派或本机均可；需能装 **Docker**（推荐）或能运行官方二进制。                                             |
| 网络         | 本机试用：访问 `http://127.0.0.1:6277` 即可。公网访问：安全组/防火墙放行 **6277**（或你映射的端口）。                      |
| 域名（可选） | 不绑定域名也能用 IP 访问；若要用 HTTPS，需在反代（Nginx、Caddy 等）里配置证书，详见 [安装部署](/docs/start/installation)。 |
| `JWT_SECRET` | **必须**改成随机长字符串，用于会话签名；不要用文档里的示例值上生产。                                                       |

**不需要**提前会写 Go 或前端；自托管用户按文档跑容器即可。

---

## 第一步：一条命令启动（Docker）

在终端执行（与 [安装部署](/docs/start/installation) 一致）：

```bash
docker run -d \
  --name ech0 \
  -p 6277:6277 \
  -v /opt/ech0/data:/app/data \
  -e JWT_SECRET="请改为随机长字符串" \
  sn0wl1n/ech0:latest
```

- `-v` 把数据存到宿主机 `/opt/ech0/data`，重装容器也不丢库；路径可按需修改。
- 若端口被占用，把左侧 `6277` 改成例如 `8080:6277`，浏览器则用新端口访问。
- 命令里的 `JWT_SECRET` 是安全关键，请替换为随机长字符串，勿照抄示例。

**打不开的常见原因**：防火墙未放行、Docker 未启动、IP 填错（云服务器要用**公网 IP** 或绑定的域名）。

---

## 第二步：浏览器里完成初始化

1. 访问 `http://<你的 IP 或域名>:6277`。
2. 按向导 **注册第一个账号**。该账号会自动成为 **Owner**（最高权限）。
3. 进入后台或设置页，确认 **JWT_SECRET** 已在部署时设为强随机值（不要在截图或 Issue 里泄露）。

当前版本下，**发帖权限通常默认只给高权限账号**；普通注册用户可能只能浏览，需由管理员在设置里授权。若你「登录了但发不了帖」，先确认是否用 Owner 登录，或查 [常见问题](/docs/start/faq)。

---

## 第三步：建议你马上做的几件事

1. **改强 `JWT_SECRET`**（若仍用示例值）：停容器 → 用新环境变量重建容器，数据卷勿删。
2. **备份数据目录**：至少复制一份 `/opt/ech0/data`（或你的挂载路径）到别处。详见 [数据管理](/docs/guide/datacontrol)。
3. **填写「服务地址」**（若用 Connect / 头像显示）：在 **系统设置 → 服务地址** 填本站完整 URL（含 `http://` 或 `https://`），与浏览器地址栏一致。
4. **规划 HTTPS**：公网站点建议用反代终止 TLS；仅内网可暂时 HTTP。

## 名词与界面

| 名词               | 含义                                                                                                 |
| ------------------ | ---------------------------------------------------------------------------------------------------- |
| **Echo**           | 一条时间线动态（类似一条微博），可含 Markdown、图片、链接卡片等。                                    |
| **实例**           | 你部署的这一套 Ech0 服务；多实例之间可用 Connect / Hub 互联，见 [互联聚合](/docs/guide/federation)。 |
| **Owner / 管理员** | 首个注册账号为 Owner；可在设置里管理用户与部分全站选项。                                             |
